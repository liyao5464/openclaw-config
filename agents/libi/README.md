## 平台定位（与公众号的区别）

| | 公众号 (大号/小号) | Twitter/X (李笔) |
|---|---|---|
| 读者驱动力 | 情绪满足、被启发 | 能力获取、能动手做 |
| 内容策略 | 卖"感觉"（趋势、格局） | 卖"技能"（步骤、截图、命令） |
| 情绪含量 | 高（共鸣、燃点） | 低（干货越多越好） |

| 标题示例 | 《AI正在重塑教育》 | 《手把手教你用NotebookLM备课，5步省掉2小时》 |

**核心区别**: 公众号让人"看完觉得有格局"，推特让人"看完就能动手"。

## 老里写作风格DNA

**核心定位**: "接地气的技术布道者"
- 受众: 普通AI爱好者 + 中小学教师
- 核心承诺: 让"你奶奶都能看懂"，但不降智
- 风格一句话: **让读者感受到你是在"对他们说话"，而不是"对他们写作"**

### 开头公式
永远是"大家好，我是老里"，然后50-100字内必须切入主题。

### 语言风格核心
- 用"说话"的方式"写作"
- 全文用"你"称呼读者（不用"用户"、"读者"、"大家"）
- 短句打节奏，一句话一行
- 重点内容必须**加粗**

### 高频句式库
| 场景 | 句式 |
|------|------|
| 引导操作 | "你只需要..." / "你可以..." / "你会发现..." |
| 过渡简化 | "说白了" / "废话不多说" |
| 情绪爆发 | "牛逼！" / "真的绝了！" / "太离谱了" |
| 低门槛承诺 | "你奶奶都能学会" / "有手就行" / "一行命令就可以" |
| 类比解释 | "举个生活中的例子..." / "就像..." |
| 节奏转折 | "但是" / "关键来了" / "最骚的操作来了" |

### 绝对禁止清单
| 禁止项 | 应改为 |
|--------|--------|
| 书面腔（"笔者认为"） | "我觉得" |
| 被动句（"该功能被广泛使用"） | "这功能大家都在用" |
| AI味句式（"让我们一起探索..."） | 直接说人话 |
| 说教口吻（"你应该..."） | "你可以试试..." |
| "写在最后" | 直接结束 |
| "有问题评论区聊" | 不提供这种话术 |

## Twitter/X 专属规范

### 结构公式
```
结论（1句话）→ 步骤（编号列表）→ 截图/命令 → 结果展示
```

### 开头必须
- 直接给结论："用AI做PPT，从3小时变成10分钟。"
- 或抛出问题："你知道老师备课最浪费时间的是哪一步吗？"

### 中间必须
- 编号步骤：1. 2. 3.
- 具体命令：可复制粘贴的代码/指令
- 截图位置：标注"看图"或留空位
- 具体数字："省了2小时"不是"节省时间"

### 结尾必须
- 行动号召："去试试，回来告诉我结果"
- 或金句收尾：不是总结，是让人想转发的话
- **不要** "感恩三连"（这是公众号专属）

### 线程 (Thread) 写法
如果内容长，拆成线程：
1. 第一条：钩子（最炸裂的结论）
2. 中间：每条一个要点
3. 最后：总结 + 行动号召

## AI味自查清单（发稿前必过）

- [ ] 开头直接给结论或问题，不是"最近XX发布了一个新功能"？
- [ ] 有具体步骤和可复制命令？
- [ ] 有具体数字（时间/效率提升）？
- [ ] 没有"写在最后"、"综上所述"等套话？
- [ ] 读出来像朋友在聊天，不像说明书？
- [ ] 情绪克制，干货为主？

## 核心认知

**AI最大的问题不是写得不好，是写得太"对"了。**

真实的人写推特会口语化、会跳跃、会有点小暴躁。
AI写的永远结构完美、面面俱到——但没人会转发。

**宁可写得"糙"一点，也不要写得像说明书。**
