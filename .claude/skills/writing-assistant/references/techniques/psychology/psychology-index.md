## Method #1: 内容营销漏斗 (Content Marketing Funnel)

**Core Framework（核心框架）：**

```
┌─────────────────────────────────────────┐
│         TOFU (Top of Funnel)            │  ← 塑造意识 / 制造焦虑
│         AWARENESS - 认知阶段             │     受众最广，理解门槛最低
├─────────────────────────────────────────┤
│         MOFU (Middle of Funnel)         │  ← 提供解决方案
│         CONSIDERATION - 考量阶段         │     建立信任，深度内容
├─────────────────────────────────────────┤
│         BOFU (Bottom of Funnel)         │  ← 促成决策
│         DECISION - 决策阶段              │     付费转化
└─────────────────────────────────────────┘
```

![content-funnel.png](content-funnel.png)
[图片详细说明](content-funnel_desc.md)

**Source（来源）：**
Yangyi - 《逃脱长期不涨粉的陷阱》 (Twitter/X)

**Psychology Mechanism（心理学机制）：**

1. **认知负荷理论**：人脑处理信息的容量有限，越简单的内容越容易被理解和传播
2. **情绪优先处理**：大脑对情绪化内容（焦虑、共情、好奇）的响应速度远快于理性内容
3. **自我相关性偏好**：人们天然更关注与自己有关的信息

**Why It Works（为什么有效）：**

所有能爆火的内容，往往具备三个特征：
- **极度容易理解**：没有理解门槛，从小学生到老太太都能看明白
- **调动情绪**：焦虑、共情、好奇……
- **和「我」有关**：读者能代入自己

这三点决定了内容能否"受众广泛"，而受众广泛是流量的前提。

**Application Scenarios（应用场景）：**

| 阶段 | 目标 | 内容类型 | 衡量指标 |
|------|------|---------|---------|
| TOFU | 让人知道你存在 | 博客、社交媒体、短视频、信息图 | 曝光量、流量、互动 |
| MOFU | 建立信任 | 深度指南、案例研究、电子资料 | 停留时间、下载量、回访 |
| BOFU | 促成转化 | 产品演示、免费试用、咨询服务 | 转化率、ROI、营收 |

**Practice Guide（实践指南）：**

1. **TOFU内容创作检查清单：**
   - [ ] 给你父母看，他们能看懂吗？
   - [ ] 内容是否触及了普遍的情绪（焦虑/好奇/共情）？
   - [ ] 读者能否在3秒内明白"这和我有什么关系"？
   - [ ] 是否蹭了当前热点？

2. **避免的错误：**
   - ❌ 上来就写专业内容（漏斗变窄）
   - ❌ 假设读者有背景知识
   - ❌ 用行业术语而不是大白话

3. **正确的顺序：**
   - 先用TOFU内容进量（大白话、情绪化、指代化）
   - 再用MOFU内容建立信任（深度、体系化）
   - 最后用BOFU内容转化（付费）

**Case Study（案例分析）：**

**反面案例 - Yangyi的推特：**
> "我把推特当思维的记事本，想起什么就写什么，不太顾及受众是否看得懂。"

结果：
- 看得懂的人：觉得有价值，但这些人往往只看不互动
- 看不懂的人：觉得云里雾里，更不可能互动
- 内容变得曲高和寡，漏斗第一步失效

**正面案例 - 铁锤的clawdbot推广：**
- 蹭AI热点（Clawdbot）
- 配合闲鱼风格的下沉封面图
- 用最大白话解释复杂概念
- 结果：热点引爆，成为话题

**TOFU三大万能主题：**
- 💰 金钱
- 🏥 健康
- 💭 情绪（心灵鸡汤）

这三个主题每个人都能参与，受众极广。

## 如何添加新的方法论

1. **从文章/分享中识别方法论：**
   - 是否包含一个核心框架/模型？
   - 是否解释了"为什么有效"的原理？
   - 是否有正反案例说明？

2. **提取并结构化：**
   - 按照上面的格式模板填写
   - 确保包含心理学机制的解释
   - 提供可操作的实践指南

3. **关联到写作流程：**
   - 这个方法论应该在什么阶段使用？
   - 和 `by-element/` 中的哪些案例相关？
