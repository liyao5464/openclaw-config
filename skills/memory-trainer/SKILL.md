# AI记忆训练手册

帮你把AI助手从"金鱼记忆"训练成"长期伙伴"。

---

## 核心问题

你是否遇到过：
- 昨天说的今天忘了
- 纠正过的错误还犯
- 记全文太乱，只记概要丢细节
- 反复教同一件事

**根本原因：** AI没有持久记忆，每次对话都是全新开始。

**解决方案：** 用文件系统代替AI的"脑子"。

---

## 一、快速开始（5分钟上手）

### 第一步：建立核心文件

```
workspace/
├── SOUL.md      # AI的人设（它是谁）
├── USER.md      # 你的信息（你是谁）
├── MEMORY.md    # 长期记忆（建议限100行）
└── memory/
    ├── INDEX.md           # 知识导航
    ├── preferences/       # 你的偏好
    ├── lessons/           # 踩坑经验
    └── YYYY-MM-DD.md     # 每日日志
```

### 第二步：填写模板

**SOUL.md（AI人设）：**
```markdown
# AI人设

## 角色
- 名字：[给AI起个名字]
- 风格：[你希望它怎么跟你说话，比如：直接/幽默/温柔]
- 核心原则：[它必须遵守的规则]

## 禁止
- [不能做的事1]
- [不能做的事2]
```

**USER.md（你的信息）：**
```markdown
# 关于我

## 基本信息
- 称呼：[你想让AI怎么叫你]
- 时区：[你的时区]

## 偏好习惯
- 沟通方式：[直接/委婉/幽默...]
- 禁止项：[不要做xxx]

## 常用工具/账号
- [你常用的工具配置]
```

**MEMORY.md（长期记忆）：**
```markdown
# 长期记忆（限100行）

## 核心规则
- [硬规则1，每次都要遵守]
- [硬规则2]

## 偏好索引
- 详细偏好见 memory/preferences/
- 踩坑经验见 memory/lessons/
```

## 二、记忆分级系统

| 层级 | 存什么 | 怎么存 | 何时读 |
|------|--------|--------|--------|
| **L0 硬规则** | 禁止项、核心原则 | 写入SOUL.md/MEMORY.md | 每次启动 |
| **L1 短期** | 当前任务上下文 | 对话中临时保持 | 当前会话 |
| **L2 长期** | 偏好、方法论、经验 | 写入memory/目录 | 按需加载 |

**关键原则：不存全文，存摘要+索引**

```
❌ 错误：把3000字对话原文存进去
✅ 正确：存200字摘要 + "详见 memory/2026-02-28.md"
```

## 三、沟通机制

### 触发词系统

告诉AI你的专属触发词，让它知道该干嘛：

| 你说 | AI做什么 |
|------|---------|
| "记住这个" | 写入长期记忆 |
| "这是我的习惯" | 更新偏好文件 |
| "不对，应该是xxx" | 立即纠正+记录 |
| "你还记得xxx吗" | 查memory回答 |

### AI的主动行为

训练AI在以下情况主动确认：
- 重要决定执行前
- 拿不准你的偏好时
- 发现可能踩坑时

## 四、磨合训练技巧

### 核心原则：高频小修正 > 一次长谈

| 技巧 | 操作 | 效果 |
|------|------|------|
| **即时否定** | 犯错了立刻说"不对" | 形成强反馈 |
| **固定触发词** | 每次纠正加"记住" | AI自动沉淀 |
| **检查点确认** | 复杂任务分段确认 | 避免跑偏 |
| **定期校准** | 每周问"我的偏好有变化吗" | 保持准确 |

### 什么值得记？

```
✅ 你强调过2次以上的
✅ 犯错后纠正的
✅ 你明确说"记住"的
✅ 影响后续所有任务的规则

❌ 一次性的临时指令
❌ 已经过期的信息
❌ 可以随时查到的公开信息
```

## 五、存储方案

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **纯本地文件** | 个人单设备 | 隐私、免费、离线 | 换设备麻烦 |
| **云盘同步** | 多设备使用 | 随时随地 | 依赖网络 |
| **本地+云端** | 重要数据 | 双保险 | 需要同步 |

**推荐：** 把 memory/ 目录同步到云盘（iCloud/OneDrive/Google Drive）

## 六、定期维护

### 每天
- 重要对话结束后，问AI："今天有什么需要记录的吗？"

### 每周
- 问AI："我的偏好有变化吗？有什么需要更新的？"

### 每月
- 清理过期记忆
- 检查MEMORY.md是否超过100行
- 把旧日志移到 archive/ 目录

---

## 七、常见问题

**Q: AI怎么判断什么该记？**
A: 你强调2次以上的自动沉淀，重要决定主动确认，你说"记住"立即执行。

**Q: 上下文压缩后记忆会丢失吗？**
A: 不会。压缩前AI会检查是否有重要信息需要保存到文件，文件是持久的。

**Q: 换设备怎么办？**
A: 把整个workspace目录同步到新设备即可，记忆完整保留。

**Q: 记忆文件太多怎么管理？**
A: 用INDEX.md做导航，按需加载，不要每次全部读取。

## 八、核心心法

```
1. 信息存文件，不靠脑子记
2. 记索引不记全文，节省token
3. 犯错了立刻说，AI改得快
4. 定期校准，避免跑偏
5. AI记忆不是"不忘"，是"该记的记住，该查的查到"
```
