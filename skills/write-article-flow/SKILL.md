# 写文章检查点工作流

一键启动写文章任务流，带检查点控制，确保每一步都经过你确认。

## 触发方式

- 你说"写文章"、"开始写"、"创建文章"、"进入写文章流程"
- 或者指定具体选题："写一篇关于 XXX 的文章"

## 工作流概览

```
启动 → 选题确认(检查点1) → 大纲确认(检查点2) → 
初稿确认(检查点3) → 去AI味(检查点4) → 微信格式(检查点5) → 完成
```

## 详细流程

### 第0步：接收任务

用户说"写文章"或指定选题。

**你问：**
> 想写什么主题？告诉我选题和大概方向。

用户给出选题后，进入检查点1。

---

### 检查点1：选题确认

**你产出：**
- 选题名称
- 目标受众
- 核心观点/角度
- 预计字数

**你问：**
> 选题这样定可以吗？有要改的吗？

**等待用户确认**，用户说"可以"、"OK"、"没问题" → 进入检查点2

用户说"改一下XXX" → 修改后重新确认

---

### 检查点2：大纲确认

**你产出：**
- 文章结构（开头、几个小节、结尾）
- 每部分的核心内容要点
- 预估字数分配

**你问：**
> 大纲这样可以吗？想调整哪里？

**等待用户确认** → 进入检查点3

---

### 检查点3：初稿确认

**你产出：**
- 完整初稿（Markdown格式）
- 建议的配图位置（标注`(配图：XXX)`）

**你问：**
> 初稿写完了，你看看怎么样？

**等待用户确认** → 进入检查点4

用户说"改第X部分" → 只改那部分 → 重新确认

---

### 检查点4：去AI味

**说明：** 这一步用子 agent 帮你改写，去掉AI写作的套路感。

**你产出：**
- 改写后的版本
- 说明改了哪些调整（口语化、删emoji、调整语气等）

**你问：**
> 去掉AI味的版本好了，你看看读起来像人写的吗？

**等待用户确认** → 进入检查点5

---

### 检查点5：微信格式转换

**你产出：**
- 转换为微信适配的HTML格式
- 封面配图建议（如果需要）

**你问：**
> 微信格式好了，要现在发布吗？还是再改改？

**等待用户确认** → 完成

---

## 产出文件

文章完成后，保存在 `/root/.openclaw/workspace/articles/` 目录：
- 原文稿：`YYYY-MM选题名.md`
- 去AI味版：`YYYY-MM选题名-v2.md`
- 微信格式：`YYYY-MM选题名-wechat.html`

## 检查点原则

1. **不跳过任何检查点**：即使时间紧，也要等用户确认
2. **每次只问一件事**：确认选题就只问选题，别问完选题又问大纲
3. **用户说"可以"才往下走**：任何犹豫都要问清楚
4. **记录用户的修改意见**：每次确认时用户改了啥，记下来

## 快捷命令

用户可以说：
- "跳过检查点" → 一次性跑完所有步骤（风险自担）
- "直接到第X步" → 从某个检查点开始
- "取消" → 中断工作流

## 注意事项

- 用 MiniMax 模型写（更适合中文写作）
- 每一步产出后用简洁的问句确认，别绕弯子
- 用户没回复时不要自动往下跑
