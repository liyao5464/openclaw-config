## 一、为什么你的AI助理"不听话"

先说一个很多人没意识到的事情。

OpenClaw 装好之后，它其实什么都不知道。

它不知道你是谁，不知道你喜欢什么风格，不知道你想让它干什么，不知道你的作息时间，不知道你的工作习惯。

**它就是一张白纸。**

你跟它说"帮我写篇文章"，它当然只能给你一篇四平八稳的东西——因为它不知道你想要什么风格，不知道你的受众是谁，不知道你的语气偏好。

这不是它的问题，是你没有给它"入职培训"。

把它当成一个刚入职的新员工。

你不告诉他公司文化、工作方式、你的偏好，他只能按照自己的理解来干活。

结果当然不是你想要的。

**解决方法很简单：把你的要求写下来，放到它能看到的地方。**

OpenClaw 的工作区里，有三个最重要的文件，就是干这个用的。

---

## 二、第一个文件：SOUL.md（告诉它是谁）

SOUL.md 是它的"灵魂文件"。

这个文件决定了它的性格、说话方式、跟你相处的风格。

**怎么编辑：**

在服务器终端里运行：

```
nano /root/.openclaw/workspace/SOUL.md
```

你会看到一个文本编辑器，把里面的内容改成你想要的。

**写什么？**

很多人不知道写什么，我直接给你一个模板：

```
你是我的私人助理，名字叫[起个名字]。

说话风格：
- 口语化，不要用书面语
- 简洁直接，不要废话
- 偶尔可以开个玩笑，但不要太过分
- 不要说"您好"、"请问有什么可以帮助您的"这种客服腔

我的偏好：
- 我喜欢直接给结论，不喜欢铺垫太多
- 回复不要太长，能用一句话说清楚就不用三句话
- 如果我问的问题你不确定，直接说不确定，不要瞎编
```

写完按 `Ctrl+X`，然后按 `Y`，然后按回车保存。

**效果对比：**

配置前，你问"今天天气怎么样"，它回答：

> 您好！根据您的位置，今天天气情况如下：温度XX度，湿度XX%，建议您……

配置后，同样的问题，它回答：

> 今天北京晴，18度，出门不用带伞。

**就这么简单，一个文件，性格全变。**

---

## 三、第二个文件：USER.md（告诉它你是谁）

USER.md 是关于你的档案。

它越了解你，帮你做事就越准。

**编辑方法：**

```
nano /root/.openclaw/workspace/USER.md
```

**写什么？**

```
基本信息：
- 名字：[你的名字]，叫我[你希望它怎么称呼你]
- 职业：[你的工作]
- 时区：北京时间（GMT+8）
- 作息：早[X]点起床，晚[X]点睡觉

工作习惯：
- 我主要用[工具/平台]工作
- 我的主要任务是[你的工作内容]
- 我不喜欢被打扰的时间：[比如午休12-14点]

偏好：
- 喜欢：[你喜欢的风格/内容]
- 不喜欢：[你不喜欢的]
- 语言：中文为主，技术词汇可以用英文
```

**一个真实的例子：**

我的 USER.md 里写了：

> 老里是AI科普博主，受众是普通AI爱好者和中小学教师。他喜欢口语化的表达，不喜欢学术腔。他的时区是北京时间，作息大概是早9点到晚12点。写文章时，受众是完全不懂技术的普通人。

有了这些，它在帮我写东西的时候，就知道该用什么风格，该考虑什么受众。

**你写得越详细，它就越懂你。**

## 四、第三个文件：HEARTBEAT.md（让它主动找你）

这是很多人忽略的一个文件，但它是OpenClaw最强大的功能之一。

HEARTBEAT.md 是它的"主动检查清单"。

配置好之后，它会定期"巡逻"，发现有需要你注意的事情，主动找你。

**编辑方法：**

```
nano /root/.openclaw/workspace/HEARTBEAT.md
```

**写什么？**

```
每天检查（轮流，不要每次都查）：
- 重要邮件：有没有需要我回复的？
- 日历事件：接下来24小时有什么安排？
- 待办事项：有没有快到期的任务？

主动提醒规则：
- 有重要邮件 → 立刻通知我
- 日历事件在2小时内 → 提前提醒
- 深夜（23点-8点）→ 不打扰，除非紧急
```

**配置前后的区别：**

配置前：你每天要主动打开AI问"今天有什么安排"。

配置后：早上8点，它主动发给你：

> 老里早上好！今天下午3点有个会议，记得准备。另外有两封邮件需要你看一下。

**你不用主动找它，它会主动找你。**

这才是AI助理应该有的样子。

## 五、怎么跟它说话，它才能真正帮到你

配置好三个文件之后，还有一个关键——

**你跟它说话的方式，决定了它能帮你多少。**

很多人跟AI说话的方式，就像发微信语音：

> "帮我弄一下那个东西"

> "你知道我说的是什么吗"

> "就是上次那个"

这种说法，人能理解，AI不行。

**跟AI说话，要说清楚四件事：**

**1. 你要什么（目标）**

不要说"帮我写篇文章"，要说"帮我写一篇公众号文章"。

**2. 给谁看（受众）**

"受众是完全不懂技术的普通人"比"给普通人看"要清楚得多。

**3. 什么风格（要求）**

"口语化，不要学术腔，2000字左右"。

**4. 有什么限制（约束）**

"不要用'首先其次最后'的结构，不要太正式"。

**一个对比例子：**

❌ 模糊版：
> 帮我写篇关于DeepSeek的文章

✅ 清晰版：
> 帮我写一篇关于DeepSeek V3的公众号文章，受众是普通AI爱好者，口语化风格，2000字左右，开头要有一个能勾住人的故事，结尾要有行动号召

同样的AI，同样的模型，清晰版的结果会好10倍。

**说得越清楚，它干得越好。**

## 六、进阶技巧：让它越来越懂你

用了一段时间之后，你可以用这个技巧让它越来越符合你的口味。

**技巧一：好的结果，让它记住**

每次它帮你做了一件事，效果很好，告诉它：

> 这个很好，把这个风格记到USER.md里，以后都这样。

它会自动更新你的档案，下次不用你再说。

**技巧二：不好的结果，告诉它哪里错了**

效果不好，不要只说"不对"，要说清楚哪里不对：

> 这个太正式了，我不喜欢这种风格。以后帮我写东西，语气要更随意一点，像朋友聊天那种。把这个记下来。

**技巧三：给它设定"禁区"**

有些事情你不想让它做，直接写进SOUL.md：

```
禁止事项：
- 不要用"综上所述"
- 不要用"值得注意的是"
- 不要用排比句（首先...其次...最后...）
- 回复不要超过200字，除非我特别要求
```

**技巧四：用定时任务替代重复指令**

如果你每天都要问同一件事，不如设置成定时任务：

> 帮我设置：每天早上8点，推送今日AI领域新闻，发到我的Telegram，不超过5条，每条一句话总结。

设置一次，以后每天自动来，不用你再问。

## 七、常见问题

**Q：我改了文件，但它还是老样子？**

需要重启才能生效：

```
openclaw gateway restart
```

**Q：它记不住我说的话怎么办？**

每次对话结束后，它的"短期记忆"会清空。

想让它记住某件事，要明确说：

> 把这个记到USER.md里

或者：

> 记住这个偏好，写进你的记忆文件

**Q：它有时候回答得很好，有时候又很差，为什么？**

这跟你的提问方式有关。

说得越清楚，结果越稳定。

如果某次回答特别好，把那次的提问方式保存下来，下次照着用。

**Q：可以同时配置多个助理吗？**

可以。OpenClaw支持多个Agent，每个有自己的性格和职责。

比如一个负责写作，一个负责查资料，一个负责提醒日程。

这个是进阶玩法，等你把基础配置搞熟了再研究。

## 避坑指南：这些坑我替你踩过了

配置 OpenClaw 的过程中，有几个地方很容易卡住，提前说清楚：

**1. 遇到不会的，直接让 OpenClaw 自己来**

安装 Skills、配置工具这些事，不用自己研究文档。直接告诉它"帮我安装 XXX"，它会自己搞定。这是它的强项，别客气。

**2. Web 界面可以切换中文**

OpenClaw 的网页端默认是英文，进设置把语言改成中文，用起来顺手很多。

**3. 全程开科学上网**

这个不用多说。很多功能依赖境外服务，没有梯子很多东西跑不起来。

**4. ClawHub 要登录才能用**

ClawHub 是 OpenClaw 的技能市场。不登录会限速，安装 Skills（比如 Twitter、YouTube 这些）也必须登录账号才能操作。

**5. 想做更多事，就得给更多权限**

OpenClaw 能做的事情，取决于你给它配置了多少"通道"：
- 要读 Twitter？配 Twitter cookies
- 要发邮件？配邮箱授权
- 要操作 Google 服务？登录 Google 账号

建议专门申请一个独立的 Google 账号给它用，和个人账号分开，更安全也更方便管理。

**6. API 配置是核心**

很多高级功能需要 API Key，比如图像生成、语音合成、搜索增强等。这些都需要你自己去对应平台申请，然后填进配置里。没有 API，很多功能就是空壳。

## 写在最后

很多人觉得AI不好用，其实不是AI的问题。

是我们还没学会怎么跟它相处。

它不是一个搜索引擎，你输入关键词它给你结果。

它更像一个人，你跟它说得越清楚，它帮你做得越好。

你给它一个清晰的目标，它能帮你跑完整个流程。

你给它一个模糊的指令，它只能给你一个模糊的结果。

**沟通是双向的，哪怕对方是AI。**

今天说的这几步，做起来其实不难：

- 花10分钟写好SOUL.md，告诉它是谁
- 花10分钟写好USER.md，告诉它你是谁
- 花5分钟写好HEARTBEAT.md，告诉它主动检查什么
- 说话的时候，说清楚目标、受众、风格、限制

就这四件事，你的AI助理会脱胎换骨。

不再是那种等你来问的工具，而是那种真正在帮你跑的伙伴。

试试吧，真的值得。
