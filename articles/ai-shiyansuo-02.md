## 先说清楚：它到底是什么

很多人第一次听说 OpenClaw，以为它是另一个 ChatGPT。

不是的。

**ChatGPT 是"会说话的工具"，OpenClaw 是"会做事的助理"。**

举个例子：

你问 ChatGPT："帮我把这篇文章发到公众号"

ChatGPT："好的，你可以这样操作：1. 打开公众号后台……"

你问 OpenClaw 同样的问题：

OpenClaw：直接调用 API，把文章转成 HTML，上传封面图，推到草稿箱。"已完成，草稿已推送，请去后台确认发布。"

**这就是区别。一个给你建议，一个帮你执行。**

OpenClaw 是开源的，部署在你自己的服务器上，数据不上传云端，你完全掌控。

## 第一关：安装（别被命令行吓到）

OpenClaw 需要用命令行安装，但其实不难。

**环境要求：**
- Node.js v18 或以上
- 一个 AI 模型的 API Key（Claude 或 GPT 都行）

**检查你的 Node.js 版本：**

```bash
node --version
```

如果显示 v18 以上，直接下一步。如果没有或版本太低，去 nodejs.org 下载安装。

**安装 OpenClaw：**

```bash
npm install -g openclaw
```

**初始化配置：**

```bash
openclaw onboard
```

这个向导会问你几个问题：用哪个 AI 模型、API Key 是什么、用哪个消息平台。

**我的建议：**
- 模型选 Claude（效果最好）
- 消息平台选 Telegram（最稳定）
- 权限先选沙盒模式（安全）

✅ 完成标准：运行 `openclaw --version` 能看到版本号

---

## 第二关：连上 Telegram（让它能跟你说话）

装好之后，你需要给 OpenClaw 一个"嘴巴"——也就是消息平台。

我用的是 Telegram，配置最简单。

**步骤：**

1. 在 Telegram 搜索 `@BotFather`
2. 发送 `/newbot`，按提示创建一个机器人
3. 拿到 Bot Token（一串字母数字）
4. 在 OpenClaw 配置里填入这个 Token

配置完之后，在 Telegram 搜索你刚创建的机器人，发一条消息，它就会回复你了。

**我踩过的坑：**

第一次配置，我把 Bot Token 填错了一个字符，死活连不上。检查了半小时才发现。

所以建议直接复制粘贴，不要手打。

✅ 完成标准：在 Telegram 给机器人发消息，它能正常回复

## 第三关：让它认识你（最关键的一步）

这一步是大多数人跳过、然后抱怨"AI 不好用"的根本原因。

OpenClaw 刚装好的时候，它什么都不知道。

它不知道你是谁，不知道你的工作是什么，不知道你喜欢什么风格，不知道你想让它怎么说话。

**你需要给它写两个文件：**

**SOUL.md — 它的人设**

告诉它：它是谁，怎么说话，什么风格。

```markdown
# SOUL.md

你是老里的内容助理。

## 工作风格
- 说话简洁，不废话
- 遇到问题直接给方案，不绕弯子
- 写文章时保持口语化，有人情味

## 禁止
- 不要用"首先、其次、最后"这种书面腔
- 不要每句话都加感叹号
```

**USER.md — 你是谁**

告诉它：你叫什么，做什么，有什么偏好。

```markdown
# USER.md

- 名字：老里
- 职业：AI科普博主
- 受众：普通AI爱好者、中小学教师
- 时区：GMT+8
- 偏好：文章要有个人经历，不要纯干货堆砌
```

这两个文件放在 `~/.openclaw/workspace/` 目录下。

**配置前后的区别：**

配置前，你问它"帮我写一篇文章"，它给你一篇标准的、没有灵魂的文章。

配置后，它知道你的受众是谁，知道你的风格是什么，写出来的东西至少有七八成是你的味道。

✅ 完成标准：给它发一条消息，感觉它"认识你"了

## 第四关：让它主动干活（定时任务）

到这一步，大多数人的用法还是"我问它，它回答"。

但 OpenClaw 真正厉害的地方是：**你不问它，它也在后台跑。**

这靠的是定时任务（Cron Jobs）。

**我设置的几个定时任务：**

**每天早上7点发早报**

```
每天早上7点，整理今天的天气、日程、AI领域最新动态，发给我。
```

**每周五下午提醒选题**

```
每周五下午3点，提醒我确认下周的文章选题。
```

**怎么设置：**

直接用自然语言告诉它：

> "帮我设置一个定时任务，每天早上7点发给我今天的天气和日程"

它会自动创建，不需要你写代码。

**我的真实感受：**

设置早报之前，我每天早上起来第一件事是刷手机，刷完脑子乱成一锅粥。

设置之后，7点准时收到一条整理好的消息，5分钟搞清楚今天要干什么。

这个改变比我想象的大。

✅ 完成标准：创建至少1个定时任务，验证它能正常触发

## 第五关：安装技能扩展（Skills）

OpenClaw 有一个技能市场，可以安装各种扩展功能。

对内容创作者来说，最实用的几个：

**天气查询**
```bash
openclaw skills install weather
```

**网页内容提取**（抓取文章、整理资料）
```bash
openclaw skills install web-reader
```

**图片生成**（封面图、配图）
根据你接入的图像 API 配置

**安装完之后怎么用：**

直接用自然语言告诉它你要做什么，它会自动调用对应的技能。

比如："帮我查一下今天北京的天气"，它会自动用天气技能查询，不需要你指定用哪个技能。

✅ 完成标准：安装至少2个技能并测试成功

## 第六关：组建你的AI团队（多Agent）

这是我觉得 OpenClaw 最酷的功能。

一个 OpenClaw 实例，可以运行多个 Agent，每个有自己的人设、记忆和工作区。

**我现在的团队配置：**

| 角色 | 职责 |
|------|------|
| 私人助理 | 统筹全局，日常沟通 |
| 内容总监 | 文章审核、公众号发布 |
| 李笔 | X/Twitter 内容创作 |
| 李微 | 微博/小红书内容 |
| AI华佗 | 健康咨询 |

每个 Agent 都有独立的 Telegram 机器人，我想找谁就找谁。

**怎么配置：**

在 `~/.openclaw/openclaw.json` 里定义 Agent 列表：

```json
{
  "agents": {
    "list": [
      { "id": "main", "name": "私人助理", "workspace": "/root/.openclaw/workspace" },
      { "id": "director", "name": "内容总监", "workspace": "/root/.openclaw/workspace-director" }
    ]
  }
}
```

每个 Agent 有自己的 workspace，里面放各自的 SOUL.md 和记忆文件，互不干扰。

**Agent 之间还能互相协作：**

我告诉私人助理"帮我把这篇文章发到公众号"，它会自动联系内容总监，内容总监负责排版和发布，完成后汇报给我。

**我的真实感受：**

以前一个人做所有事，现在感觉真的有一个小团队在帮我。

不是比喻，是真实的工作流变化。

✅ 完成标准：配置至少2个 Agent，能分别对话

## 我现在的完整工作流

## 常见问题

**Q：API 费用大概多少？**

轻度使用（每天10-20次对话）：约 $5-10/月

内容创作者（每天写稿+定时任务）：约 $20-30/月

省钱技巧：日常任务用便宜的模型（MiniMax、Kimi），重要任务才用 Claude。

**Q：需要有服务器吗？**

不一定。本地电脑也能跑，但服务器的好处是24小时在线，定时任务不会因为电脑关机而中断。

我用的是腾讯云轻量应用服务器，最便宜的配置够用，一个月几十块钱。

**Q：技术要求高吗？**

会用命令行就够了。不需要会编程。

配置文件是 JSON 格式，看起来有点吓人，但其实改起来很简单，而且可以直接让 OpenClaw 帮你改。
