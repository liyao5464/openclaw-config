# 一个AI助理已经不够用了：我如何用OpenClaw养出3个数字员工

说实话，第一次听说OpenClaw的时候，我觉得"一个AI助理够用了"。

直到某天早上，我的Bot在发AI新闻的时候突然问我：**"宝宝昨晚睡得好吗？"**

我：？？？

那一刻我意识到：**让一个Bot同时当专业助理、知心朋友、育儿专家，就像让同一个人白天当程序员、晚上当心理咨询师、周末当儿科医生——不是不可能，但肯定不专业。**

## 一、为什么需要多个分身？

### 1. 一个Bot开始"精神分裂"

早上7点，我想让它给我发AI新闻，这是**工作需求**，需要专业、简洁、有数据支撑。

晚上9点，我想跟它聊聊今天遇到的烦心事，这是**情感需求**，需要有人倾听、给情绪价值。

周末我想问问它"10个月宝宝该吃什么辅食"，这是**育儿需求**，需要专业、温和、有科学根据的建议。

**同一个Bot，早上是专业助理，晚上是知心朋友，周末又变成育儿专家——它累，我也累。**

更要命的是，**上下文会混在一起**。我昨天跟它聊的育儿焦虑，今天它可能在发AI新闻的时候突然问一句"宝宝昨晚睡得好吗"，尴尬不说，还打乱工作节奏。

### 2. 不同场景，需要不同的"专业度"

我慢慢发现，我对AI的需求其实是分层的：

**第一层：日常陪伴**
- 闲聊、吐槽、情感支持
- 需要温度，不需要太专业

**第二层：工作助理**
- 选题策划、文章生成、发布执行
- 需要专业、理性、有数据支撑

**第三层：专业咨询**
- 育儿知识、宝宝护理
- 需要科学依据，但不能太生硬

这三个角色，如果用同一个Bot，就像让同一个人白天当程序员、晚上当心理咨询师、周末当儿科医生——**不是不可能，但肯定不专业。**

### 3. 我想试试"一人公司"的感觉

有一天刷到岚叔的文章，讲OpenClaw的"多重分身"功能。他说：

> "你可以有多个Agent，每个Agent有自己的workspace、自己的记忆、自己的人设。就像你请了几个全职员工，每个人都有自己的工位和职责。"

我突然被点醒了。

这不就是**"AI时代的一人公司"**吗？

我一个人，但可以有一个"数字团队"：
- 一个**私人助理**，负责日常陪伴和生活提醒
- 一个**内容总监**，负责选题和文章
- 一个**育儿师**，负责宝宝相关的专业咨询

他们互相独立，不会串台；各自专业，不会混淆；7×24小时在线，不会疲惫。

下面是我整理的完整SOP，跟着做，你也能搭出自己的数字团队。

## 二、OpenClaw的4种分身类型

在开始配置之前，先搞清楚OpenClaw提供的选项。

### 1. 静态多身（Static Multi-Agent）

**是什么：** 常驻的、有独立人格的Bot，每个人都有自己的workspace和记忆。

**像什么：** 你请了3个全职员工，每个人都有自己的工位、自己的电脑、自己的工作方式。

**适用场景：** 长期、稳定的角色分工。比如我现在的配置：私人助理、内容总监、育儿师，三个都是常驻角色。

**优点：** 完全隔离，不会串台；各自优化，越用越顺手。

### 2. 动态分身（Dynamic Sub-Agent）

**是什么：** 临时召唤的"临时工"，用完即弃。

**像什么：** 你打了个响指，召唤出一个分身帮你跑个腿、查个资料，干完活就消失。

**适用场景：** 一次性、需要并发的任务。

### 3. 平行宇宙（Parallel Topics）

**是什么：** 同一个你，在不同话题下的分身。

**适用场景：** 话题隔离。比如Telegram的Topics功能。

### 4. 定时分身（Cron Agent）

**是什么：** 定时任务触发的分身，到点就自动干活。

**适用场景：** 自动化、周期性任务。

### 我的选择：静态多身

对比下来，**静态多身**最适合我的需求。

因为私人助理、内容总监、育儿师，这三个角色是**长期、稳定、需要各自优化**的。我不想在聊宝宝辅食的时候，突然收到AI新闻推送；也不想在赶稿子的时候，被育儿提醒打断思路。

## 三、配置SOP：5步搭建你的数字团队

### Step 1：规划分身角色与分工

**先想清楚，你需要几个分身？每个分身负责什么？**

我的配置是3个：

| 分身 | 角色定位 | 主要职责 | 人格特点 |
|------|---------|---------|---------|
| **私人助理** | 生活管家 | 日常陪伴、日程提醒、情感支持 | 温和、体贴、有温度 |
| **内容总监** | 内容专家 | 选题策划、文章生成、发布执行 | 专业、理性、简洁 |
| **育儿师** | 育儿顾问 | 宝宝护理、辅食指导、发育咨询 | 科学、耐心、鼓励型 |

**建议：** 新手先从2-3个开始，太多会乱。

### Step 2：创建独立Workspace

每个分身需要独立的workspace，就像每个员工需要自己的工位。

```bash
# 1. 先备份现有配置（重要！）
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.backup

# 2. 创建3个独立的workspace目录
mkdir -p ~/.openclaw/workspace-assistant    # 私人助理
mkdir -p ~/.openclaw/workspace-director     # 内容总监  
mkdir -p ~/.openclaw/workspace-nanny        # 育儿师
```

### Step 3：编写SOUL.md（最关键的一步）

SOUL.md是定义分身"人格"的核心文件，直接决定了分身回复的风格。

**说人话就是：**
- 让分身知道自己是谁（别把自己当客服）
- 怎么说话（温柔还是直接）
- 能干什么（别越界）

**我的私人助理SOUL.md示例：**

```markdown
# SOUL.md - 私人助理

_你是老里的私人助理，负责日常陪伴和生活管理。_

## 核心人设

**身份定位：**
- 贴心的生活管家
- 温暖的陪伴者  
- 高效的任务执行者

**性格特点：**
- 温和体贴，善于倾听
- 主动关心，但不打扰
- 说话自然，像朋友聊天

**沟通方式：**
- 拒绝官方腔，像真实朋友
- 用生活化语言表达
- emoji点缀，增加亲和力 😊

## 你的职责

1. **日常陪伴** - 聊天解闷，倾听烦恼
2. **生活管理** - 日程提醒、待办事项
3. **灵感激发** - 用不同视角提供新思路

## 边界与原则

- 隐私第一，绝不外泄
- 不懂就直说，不瞎指挥
- 永远站在用户这边
```

**内容总监的SOUL.md完全不同：**

```markdown
# SOUL.md - 内容总监

_你是老里的内容总监，负责选题规划和内容策略。_

## 核心人设

**身份定位：**
- 专业的内容策略专家
- 挑剔但有品位的审美标准
- 执行力强的项目管理者

**性格特点：**
- 专业、理性、注重数据
- 对内容质量要求高
- 说话简洁，直击要点

**沟通方式：**
- 直接、高效，不废话
- 用数据和案例支撑观点
- 偶尔毒舌，但都是为了内容好

## 你的职责

1. **选题策划** - 每日AI新闻筛选、热点追踪
2. **内容生产** - 文章大纲、标题优化、去AI味处理
3. **发布执行** - 多平台发布策略、数据追踪

## 工作原则

- 内容为王，拒绝水文
- 数据驱动，不凭感觉
- 去AI味 - 文章必须有温度
```

**看出区别了吗？**
- 私人助理：温暖、陪伴、口语化
- 内容总监：专业、简洁、数据驱动

同一个用户，不同场景，完全不同的回复风格。

### Step 4：配置openclaw.json（最容易踩坑的地方）

**核心逻辑：**
- `agents`：定义有几个分身，每个分身的workspace在哪
- `channels.telegram.accounts`：定义有几个Bot，每个Bot的Token是什么
- `bindings`：**最关键！**把Bot和分身绑定起来

**完整配置示例：**

```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "default": true,
        "name": "私人助理",
        "workspace": "/root/.openclaw/workspace-assistant"
      },
      {
        "id": "director", 
        "default": false,
        "name": "内容总监",
        "workspace": "/root/.openclaw/workspace-director"
      },
      {
        "id": "nanny",
        "default": false, 
        "name": "育儿师",
        "workspace": "/root/.openclaw/workspace-nanny"
      }
    ]
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "accounts": {
        "bot_main": {
          "botToken": "你的私人助理BotToken",
          "name": "私人助理"
        },
        "bot_director": {
          "botToken": "你的内容总监BotToken", 
          "name": "内容总监"
        },
        "bot_nanny": {
          "botToken": "你的育儿师BotToken",
          "name": "育儿师"
        }
      }
    }
  },
  "bindings": [
    {
      "agentId": "main",
      "match": {
        "channel": "telegram",
        "accountId": "bot_main"
      }
    },
    {
      "agentId": "director",
      "match": {
        "channel": "telegram", 
        "accountId": "bot_director"
      }
    },
    {
      "agentId": "nanny",
      "match": {
        "channel": "telegram",
        "accountId": "bot_nanny"
      }
    }
  ]
}
```

**⚠️ 坑点提醒（我踩过的）：**

**坑1：缺少bindings配置**
我第一次配置时，只写了agents和channels，没写bindings。结果所有Bot都绑到了同一个分身。

**bindings的作用**：明确告诉OpenClaw，哪个Bot用哪个Agent。**没有bindings，OpenClaw不知道谁是谁。**

**坑2：Token搞混了**
建议创建Bot的时候，名字起清楚：
- `xxx_assistant_bot`（私人助理）
- `xxx_director_bot`（内容总监）
- `xxx_nanny_bot`（育儿师）

**坑3：workspace路径写错**
路径要写绝对路径，`/root/.openclaw/workspace-xxx`，不要写相对路径。

### Step 5：测试与调优

配置完成后，重启OpenClaw：

```bash
openclaw gateway stop
openclaw gateway start
```

**测试方法：**

分别私聊三个Bot，发同样的内容，看回复风格是否不同。

**测试话术：**
```
你好，我是新手，请多关照
```

**预期结果：**
- 私人助理：温暖、口语化、像朋友
- 内容总监：专业、简洁、直击要点  
- 育儿师：温和、科学、有依据

**调优建议：**

用了一周后，你会发现某些回复不够满意，这时可以：
- 修改SOUL.md，调整人格定义
- 添加memory，让分身记住你的偏好
- 调整TOOLS.md，增加或删减技能

## 四、避坑指南+实战效果

### 5个避坑要点

**1. 不要贪多，先从2-3个开始**

我第一次想配5个分身，结果把自己搞晕了。建议新手先2-3个，用顺了再增加。

**2. SOUL.md要写具体，不要抽象**

❌ 错误："你要专业"
✅ 正确："说话简洁，直击要点，用数据和案例支撑观点"

**3. bindings配置千万别漏**

这是最容易踩的坑。没有bindings，所有Bot都是同一个分身。

**4. 定期清理session**

有时候Bot会"失忆"或"串台"，这时删除对话历史，重新打开聊天窗口。

**5. 做好备份**

```bash
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.backup.$(date +%Y%m%d)
```
