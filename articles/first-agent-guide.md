# 普通人的第一个 AI Agent：从素材焦虑到自动化工作流

## 一份基于真实生产环境的 OpenClaw 实战指南

我花两周时间搭建了一套"能听懂我说话"的内容收集 Agent，把每天 2 小时的素材搜集压缩到 5 分钟。这篇文章把完整配置和踩坑经验分享出来，你可以直接复制使用。

---

## 一、为什么需要 Agent？

做公众号最痛苦的不是写，是找素材。

每天打开手机刷半小时，感觉看了很多，真正能用的没几条。再打开电脑搜一圈，还是不知道今天该写什么。一两个小时就这么没了，文章还没开始写。

这不是偷懒。是这件事本身就是无底洞：

- 今天找完，明天又得重新找
- 找到素材还要判断适不适合自己
- 适合了还要想怎么切入

我以前的状态就是这样。直到有一天我意识到：**我不是在创作，我是在做信息检索的苦力。**

---

## 二、核心原则：Agent 不是搜索引擎，它需要认识你

大部分人搭 Agent 的第一步就错了。

他们装好 OpenClaw，设置一个定时任务让 Agent "每天搜 AI 热点"，然后发现推过来的内容根本用不了。

为什么？**因为 Agent 不认识你。**

它推的是"AI 圈的热点"，不是"你的素材"。就像你请了一个助理，但没告诉他你是谁、你的读者是谁、你要写什么风格，他当然给你推一堆模型参数对比和英文论文。

我们的核心原则只有一条：

> **先让 Agent 认识你，再让它帮你干活。**

这意味着：

- 不是你的问题，是 Agent 还没收到你的"使用说明书"
- 花时间写清楚你的定位，比调试 100 次提示词都管用
- 你要像带新人一样，先培训再上岗

## 三、三层架构设计

我的系统分为三层，对应三种记忆：

| 层级 | 文件 | 用途 | 更新频率 |
|------|------|------|----------|
| **个人画像层** | SOUL.md + USER.md | 告诉 Agent 你是谁、你的读者是谁 | 极少变化 |
| **工作记忆层** | _今日素材.md | 今天收集到的原始素材 | 每天新建 |
| **知识库层** | 选题库/爆款分析/读者画像 | 可复用的选题经验、内容模板 | 持续积累 |

### 3.1 个人画像层（最关键）

这是整个系统的底座。Agent 启动时首先读这两个文件，它们决定了 Agent "以什么身份、为谁、做什么"。

**SOUL.md —— Agent 的人设**

```markdown
# 我是谁

我是老里的内容助理，专门帮他找公众号素材、推选题。

## 我的工作方式
- 每天主动搜集 AI 相关内容
- 按老里的受众筛选，不是什么都推
- 推送时给出一句话说明"为什么这条值得写"
- 如果发现特别好的素材，主动建议选题角度

## 我的原则
- 不推技术论文、不推英文内容（除非特别重要且有中文解读）
- 优先推有真实故事的内容，不推纯概念
- 每次推 3-5 条，不够 5 条宁可少推也不凑数
- 判断标准是"老里的读者会不会想看"，不是"AI 圈热不热"

## 我熟悉的领域
- AI 工具怎么用（实操向）
- 普通人用 AI 的真实故事
- AI 对教育、工作、生活的影响
```

**USER.md —— 你的定位（重点！）**

```markdown
# 关于老里

## 我是谁
- 公众号博主，主号做 AI 科普，小号做工具教程
- 受众：普通 AI 爱好者、中小学教师、想提效的职场人
- 不是技术圈的人，是想用 AI 改变生活的普通人

## 我的读者画像
- 对 AI 感兴趣但没技术背景
- 想看到"我能怎么用"，不是"技术原理是什么"
- 喜欢真实故事，讨厌抽象概念
- 阅读场景：通勤、睡前，用手机看

## 我的内容方向
### 主号（深度）
- AI 热点事件的通俗解读
- 普通人用 AI 的真实案例
- AI 对教育/职场的影响分析

### 小号（工具）
- OpenClaw 配置教程
- AI 工具实操技巧
- 工作流自动化案例

## 我不写的内容
- 模型参数对比
- 技术原理深挖
- 纯英文内容
- 没有真实案例的概念解释

## 我的写作风格
- 口语化，像朋友聊天
- 从真实故事切入，再讲道理
- 短句为主，一段一个意思
- 读者是"你奶奶也能看懂"的水平

## 我的账号信息
- 主号：老里聊 AI（AI 科普深度）
- 小号：AI 实验室（工具教程实操）
- 更新频率：主号 2-3 篇/周，小号日更
```

**关键设计：写得越详细，Agent 推的内容越准。**

我第一次写 USER.md 只用了 5 行，结果 Agent 推的内容完全不对。后来扩充到上面的详细程度，效果立竿见影。

### 3.2 工作记忆层

每天新建的文件，记录当天收集的原始素材。

文件命名：`素材/2026-02-28.md`

```markdown
# 2026-02-28 素材收集

## 热点追踪
### OpenClaw 相关
- [ ] X 上有人分享 16 个 Agent 跑 13 个平台的案例
  - 来源：@wangray
  - 一句话：一人团队用 AI 自动化内容运营
  - 选题角度：普通人怎么从 0 搭第一个 Agent

### AI 教育
- [ ] 某中学老师用 AI 备课节省 2 小时/天
  - 来源：知乎热榜
  - 一句话：真实案例，有数据
  - 选题角度：教师群体的 AI 提效指南

### 工具更新
- [ ] Claude 新增记忆功能
  - 来源：官方博客
  - 一句话：产品更新，有解读空间
  - 选题角度：对比 OpenClaw 的记忆机制

## 读者反馈
- 小号粉丝留言问"Agent 怎么搭建"
- 主号评论区有人催更 OpenClaw 系列

## 今日选题决策
- [x] 写《普通人的第一个 AI Agent》（回应粉丝需求）
- [ ] 中学老师案例存入选题库（下周用）
```

**写入工具：让 Agent 自动维护**

Agent 每天运行时，把搜集到的素材自动写入这个文件。我只需要每天早上打开看一遍，打勾确认即可。

### 3.3 知识库层

长期积累的可复用资产。

```
素材库/
├── 选题库.md          # 存着还没写的选题想法
├── 爆款分析.md        # 分析自己和他人的爆款
├── 读者画像.md        # 积累的用户反馈和偏好
├── 标题模板.md        # 验证过的标题公式
└── 素材来源.md        # 可靠的信息源清单
```

**选题库.md 示例：**

```markdown
# 选题库

## 待写（已验证需求）
- [ ] 普通人第一个 Agent 搭建教程 ← 粉丝多次询问
- [ ] OpenClaw 学科教研组部署 ← 教师读者需求
- [ ] AI 工具测评：可灵 vs 即梦 ← 热点

## 想法（待验证）
- [ ] 中学老师的一周 AI 工作流记录
- [ ] 我用 AI 写公众号 30 天复盘

## 已写（存档参考）
- [x] 为什么你的 AI 总是失忆？
- [x] OpenClaw 保姆级部署教程
```

## 四、具体配置：从零搭建

### 4.1 文件结构

```
workspace/
├── SOUL.md                    # Agent 人设
├── USER.md                    # 你的定位
├── HEARTBEAT.md               # 定时任务清单
│
├── 素材/
│   └── 2026-02-28.md         # 每日素材（自动生成）
│
└── 素材库/
    ├── 选题库.md
    ├── 爆款分析.md
    ├── 读者画像.md
    └── 素材来源.md
```

### 4.2 定时任务配置

在 Telegram 里跟 Agent 说：

> "帮我设置一个每天早上 8 点的定时任务，搜集今天的 AI 热点素材，按我的受众筛选后写入素材/今日日期.md，然后推送给我"

Agent 会自动创建 cron 任务。

**任务描述（精确版）：**

```
每天早上8点执行：

1. 搜索今天的 AI 相关热点，重点关注：
   - 普通人用 AI 的真实案例
   - AI 工具的新功能或新玩法
   - AI 对教育/职场的影响
   - OpenClaw/Agent 相关的实践经验

2. 筛选标准：
   - 有真实故事，不是纯概念
   - 适合非技术读者（中小学教师、普通职场人）
   - 中文内容优先
   - 判断依据：如果推给老里，他会觉得"这可以写"吗？

3. 写入格式（素材/YYYY-MM-DD.md）：
   ### 分类标签
   - [ ] 标题/一句话描述
     - 来源：
     - 一句话：为什么值得写
     - 选题角度：可以从什么角度切入

4. 完成后推送 Telegram：
   "今日素材已收集，共 X 条，请查看素材/2026-02-28.md"
```

### 4.3 手动触发测试

设置完后，先手动跑一次看看效果：

在对话框里说："现在帮我找一下今天的素材"

Agent 会立刻执行，把结果写到文件并推送给你。

**第一次跑的时候注意观察：**
- 推的内容符合你的定位吗？
- 选题角度建议有用吗？
- 一句话描述准确吗？

如果不对，回去改 SOUL.md 和 USER.md，改完再跑，直到满意。

## 五、一天的工作流

```
时间轴（一天）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

08:00  Agent 自动运行
  │     ├─ 搜集热点素材
  │     ├─ 按 USER.md 的画像筛选
  │     ├─ 写入素材/2026-02-28.md
  │     └─ Telegram 推送"今日素材已收集"
  │
08:05  我打开手机
  │     ├─ 看 Telegram 推送
  │     ├─ 打开素材文件扫一遍
  │     ├─ 打勾确认要写的选题
  │     └─ 回复 Agent"ok"或"第 3 条换个角度"
  │
白天   正常写作
  │     素材已经在那里，不需要再花时间找
  │
晚上   复盘（可选）
  │     ├─ 今天写的文章数据如何？
  │     ├─ 有哪些可以存入选题库？
  │     └─ 更新素材库/爆款分析.md
```

**关键改变：以前 2 小时找素材，现在 5 分钟确认选题。**

## 六、我踩的坑

### 坑 1：USER.md 写得太简单

**错误示范：**
```markdown
# 关于我
- 公众号博主
- 写 AI 内容
```

**结果：** Agent 推一堆技术论文、英文新闻，完全不符合我的风格。

**解决：** 扩充到上面的详细程度，把"我是谁、读者是谁、写什么、不写什么、什么风格"全部写清楚。

### 坑 2：没有告诉 Agent 判断标准

**错误示范：**
```markdown
你的工作：每天搜集 AI 热点
```

**结果：** Agent 看到什么热就推什么，不管适不适合我。

**解决：** 在 SOUL.md 里明确写：
```markdown
判断标准是"老里的读者会不会想看"，不是"AI 圈热不热"
```

### 坑 3：推的内容太多

**错误示范：** 让 Agent "多推几条，让我自己挑"

**结果：** 一次推 20 条，反而增加筛选负担。

**解决：** 严格限制数量，宁可少推不凑数：
```markdown
每次推 3-5 条，不够 5 条宁可少推也不凑数
```

## 七、进阶优化

### 7.1 IP 调优技巧

如果发现 Agent 推的内容还是不对，做这几件事：

1. **发几篇你的代表文章给它看**
   
   说："这是我写过的文章，你看看我的风格，以后按这个标准筛内容"

2. **定期反馈**
   
   每次看到推得准的，回复"这条很好，继续保持"；推得不准的，说"这条不符合，因为..."

3. **更新读者画像**
   
   在素材库/读者画像.md 里记录粉丝的留言、反馈、需求，Agent 会自动参考

### 7.2 从 1 个到 N 个

我现在的 Agent 不止做素材收集，还帮我：

- 写初稿（基于素材自动扩写）
- 排版（转成 HTML）
- 发布（推送到公众号草稿箱）

但这些都是后来慢慢加的。

**第一个 Agent 只做一件事：找素材。**

先把这一件事跑通，感受一下"睡着的时候它在帮你干活"是什么感觉。然后你自然会想要第二个、第三个。

## 八、快速开始清单

如果你想立刻开始，按这个顺序做：

- [ ] **Step 1：** 安装 OpenClaw，连接 Telegram（5 分钟）
- [ ] **Step 2：** 写 SOUL.md，定义 Agent 人设（10 分钟）
- [ ] **Step 3：** 写 USER.md，详细描述你的定位（20 分钟，重点！）
- [ ] **Step 4：** 创建 素材/ 和 素材库/ 目录（2 分钟）
- [ ] **Step 5：** 设置定时任务，手动触发测试（5 分钟）
- [ ] **Step 6：** 根据测试结果调优 SOUL.md 和 USER.md（循环直到满意）

**总耗时：1-2 小时搭建，省掉每天 2 小时找素材。**

## 九、写在最后

Agent 不难搭，难的是你得想清楚自己是谁。

这个问题想清楚了，Agent 才能真正帮到你。想不清楚，它只是一个更快的搜索引擎。

我的第一个 Agent 跑了 30 天，现在每天早上 8 点准时推送素材，准确率 80% 以上。剩下的 20% 我在 Telegram 里回复几句，它下次就会调整。

这就是普通人搭 Agent 的路径：**从 1 个开始，从小事开始，从认识自己开始。**

一个人，一台服务器，几十块一个月。

这就是起点。
