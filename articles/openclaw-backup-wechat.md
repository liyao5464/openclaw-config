# 龙虾把自己"玩死了"？我用这两个方法，再也不怕OpenClaw崩溃

大家好，我是老里。

今天有个读者给我发消息。

说他的OpenClaw崩了。

调好的工作流、几个精心配置的Agent——全没了。

我看完，后背一凉。

因为我亲身经历过。

---

刚开始用OpenClaw那会儿，我花了好几天把龙虾调教好。

写作风格、发布流程、Agent分工……一点一点喂进去，终于顺手了。

然后有一天，龙虾把自己"玩死了"。

具体原因到现在都没搞清楚。可能是它自己改了什么配置，可能是版本冲突，也可能就是单纯的抽风。

怎么都找不到原因。

那种感觉，就像你花了一个月驯好一匹马，早上起来发现它不见了。

从那以后，我开始认真对待备份这件事。

---

所以今天不写别的。

就教你怎么给OpenClaw做自动备份。

两步，真的只要两步。

---

**第一步：搞一个GitHub私有仓库 + PAT**

打开 github.com，右上角头像 → Settings。

拉到最底部，找到 Developer settings。

左边点 Personal access tokens → Tokens (classic)。

新建一个：
- Note 随便填，比如"OpenClaw备份"
- Expiration 选90天（别选永久，安全）
- Scopes **只勾 repo**，其他不用管

生成之后，**立刻复制**。

这个 token 只显示一次，关掉页面就没了。

复制完，回到 GitHub 首页，新建一个私有仓库（Private）。

记下仓库地址，类似这样：
```
https://github.com/你的用户名/你的仓库名
```

第一步完成。

---

**第二步：告诉OpenClaw装好自动备份**

回到你的OpenClaw对话框，发这句话：

> 安装gitclaw用于在我的私有github库地址：https://github.com/你的用户名/你的仓库名 Use this PAT: ghp_你的token 创建 .gitignore 排除 .env, tokens等敏感文件

它会自动帮你配好。

配完之后问它一句："gitclaw已安装吗？"

确认没问题，搞定。

---

以后你的所有对话记录、Agent配置、工作流文件，都会**每小时自动备份**到这个私有仓库。

有改动才推，没改动不动。

安静，省心，不烧流量。

---

如果真的不幸崩了，怎么恢复？

更简单。

重新装好OpenClaw之后，一行命令：

```bash
git clone https://github.com/你的用户名/你的仓库名.git ~/.openclaw/workspace
```

全部回来。

就这么快。

---

**方法二：腾讯云快照回滚（救了我好几次）**

如果你的OpenClaw跑在腾讯云服务器上，还有一个更暴力的方法。

腾讯云有个功能叫"快照"。

简单说就是给整台服务器拍一张照片，系统、配置、文件全部定格在那一刻。

崩了？直接回滚到上一张快照。

不用找原因，不用排查问题，直接穿越回去。

我自己用这个方法救过好几次场。

**怎么操作：**

登录腾讯云控制台 → 云服务器 → 找到你的实例 → 左边菜单"快照" → 创建快照。

建议定期更新一次快照，比如每次大改配置之前先拍一张。

崩了的时候，找到最近的快照，点"回滚"，等几分钟，回来了。

**注意：** 快照回滚会把整台服务器恢复到拍照时的状态，快照之后的改动会丢失。所以两个方法最好配合用——GitHub备份保文件，快照保系统环境。
