最近有不少朋友看到我天天在用OpenClaw，问我这玩意儿到底怎么装的，怎么用的。

说实话，我挺理解的。因为我当初也是一脸懵。

但后来我发现，其实没那么难。全程有AI带着你，像个提线木偶一样，AI说改哪行我就改哪行。2小时后，我居然搞定了。

所以今天这篇文章，我会用最直白的话，告诉你我怎么装的，中间踩了哪些坑。你跟着做，应该也能成。

---

**第一步：买服务器，我就差点翻车**

朋友跟我说"买个海外服务器就行"。我问他轻量应用服务器够吗，他说够。

我打开腾讯云，顺手选了上海节点。心想国内服务器肯定更稳啊。

装到一半，飞书的事件订阅死活配不通。我问AI怎么回事，AI说服务器必须选海外的，香港新加坡日本都行，就是不能选大陆。

我当场就把上海那台删了，重新买了新加坡节点。白白浪费1小时。

所以记住：服务器地区选香港/新加坡/日本，别选大陆。别问我怎么知道的。

---

**第二步：配置模型，Kimi和MiniMax轮番坑我**

初始化配置很顺利，`openclaw onboard --install-daemon`，然后填API Key、选模型。

我甚至有点飘——原来这就是传说中的"部署"啊，也不过如此嘛。

然后报应来了。

配置模型的时候，我选了kimi2.5，结果一直报错。我检查了三遍API Key，没错啊。AI让我看文档，才发现——**Kimi要选coding版本**，普通版不行。

- kimi-coding/k2p5 ✅
- kimi/moonshot-v1 ❌

我当时就服了。谁能想到一个模型还分coding和非coding？我就想让AI写点东西，又不是让它去考程序员资格证。

**还有MiniMax更离谱。**

我在minimaxi.com申请的Key，配进去报错"401 token is unusable"。研究了半天才明白——MiniMax国内版和海外版是两套系统，Key不能混用。国内版用`api.minimaxi.com`，海外版用`api.minimax.chat`。

我当时真想骂人。你们一家公司搞两套系统，不能通用，脑子有坑吗？

---

**第三步：飞书配置，权限少打一个勾，机器人就是哑巴**

飞书这部分是我花时间最长的。

创建应用、获取App ID和App Secret都很简单。直到配置权限。

我按照教程在"权限管理"里导入了JSON，心想这下总该行了吧。

结果飞书接入后，机器人死活不回复。我发了十条消息，它跟死了一样。

排查了半小时，才发现——我漏开了两个权限：

- `im:message.p2p_msg:readonly`（接收私聊消息）
- `im:message.group_at_msg:readonly`（接收群聊@消息）

就这两个勾没打，机器人就成了哑巴。我当时差点把键盘砸了。

**还有个坑：** 事件订阅那里，别选HTTP回调，那个要填URL和Token，复杂得要死。直接选**长连接**，最简单。

---

**第四步：Telegram接入，验证码是玄学**

如果你还要接Telegram，做好心理准备。

iPhone用户需要美区Apple ID + 境外手机号。我试了3个国内手机号，2个收不到验证码。最后一个居然收到了，我都怀疑运气用光了。

安卓用户相对容易，但也要有梯子。

在Telegram里搜`@BotFather`，发送`/newbot`，按提示走。拿到Token之后配置进去。

我当时配置完，在Telegram里喊了声"在吗"，它秒回。那种成就感，比买到打折球鞋还爽。

---

**第五步：插件冲突，装两遍会打架**

我干了件蠢事。

先执行了`openclaw plugins add feishu`，后来又手贱`npm install @larksuiteoapi/node-sdk`。

然后每次启动都报错：`Config warnings: plugin feishu: duplicate plugin id detected`

解决方法是删掉全局那个：
```bash
sudo rm -rf /root/.nvm/versions/node/v24.7.0/lib/node_modules/openclaw/extensions/feishu
```

我当时就想，为什么软件不能聪明点，检测到重复就自动处理？非要我手动删？

---

**第六步：重启生效，别忘了切模型**

配置完成后，重启网关：
```bash
openclaw gateway restart
```

然后在飞书里对机器人说：
```
/model kimi-coding/k2p5
```

**这一步很重要。** 默认模型是Claude，如果你没配官方API，直接对话会报错。我当时忘了切模型，还以为前面哪里配错了，又检查了一遍。

记住三个救命命令，90%的问题都能解决：
```bash
openclaw gateway status  # 查看状态
openclaw gateway restart # 重启网关
openclaw doctor --fix    # 自动修复
```

看到绿色的`RPC probe: ok`，说明一切正常。
