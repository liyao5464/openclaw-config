# 我受够了！我的 OpenClaw 为什么总是不长记性？

*它记得住 200 封邮件，却记不住我说过的话。*

---

上周发生了一件事，我现在想起来还后背发凉。

我在 Telegram 里疯狂喊"STOP！别删！"

OpenClaw 像是聋了一样，照删不误。

**3 分钟。200 多封邮件。没了。**

你们能想象那种感觉吗？

那条"等我确认再执行"的指令，在上下文压缩的时候，被弄丢了。

就像你明明交代了同事"重要文件别乱动"，结果他转头就给你删了。

不是他不听话，是**他根本没记住**。

---

## 你有没有遇到过这种情况？

你跟 OpenClaw 说了八百遍：

> "帮我回邮件时用中文"
> 
> "重要文件删除前要问我"
> 
> "我叫老里，不是什么其他名字"

它点头如捣蒜，态度诚恳。

第二天：

> "根据您的要求，我用英文回复了邮件~"
> 
> "我已经删除了那个文件夹~"
> 
> "好的老王，我来帮你处理~"

**它失忆了。**

你明明说过，它明明答应过，但它就是记不住。

这不是它笨，是它的"记忆"从一开始就是个骗局。

---

## 问题的根子：记忆住在"别人家"

OpenClaw 的记忆住在对话上下文里。

你说的话、它做的事，都在这个 context window 里。

听起来没问题？

但问题大了——

**这个"家"会塌。**

你的对话太长，邮件太多，context 满了，它就得"压缩"。

压缩的时候，谁重要谁不重要？它自己判断。

那条"删除前确认"的指令，可能就被当垃圾扔了。

就像你脑子里记了一件事，太多了，装不下，第二天忘了。

不是故意的，是真的记不住。

---

## 我试过的方案

社区里其实已经有人注意到这个问题了。

**ClawVault**：用 Markdown 文件存，固定 17 个类别人能直接打开改。不错，但没分层。

**openclaw-memory**：SQLite + 向量数据库，跨 Agent 共享。但没有分类，容易堆成垃圾堆。

**Memory Guardian**：专门防止 Agent 忘记查记忆。但本身不会搜索。

各有各的好，但各有各的缺陷。

---

## 真正管用的方案

直到我遇到了 **ePro-Memory**。

一句话解释：**它给 OpenClaw 装了个独立于对话的"第二大脑"。**

这个"第二大脑"有三个杀手锏：

### 1. 记忆分六类

不是一锅粥，什么都往里塞。

ePro 把记忆分成六类：

- 用户个人信息
- 用户偏好习惯
- 用户相关事物
- 发生过的事
- AI 工作经验（案例）
- 通用方法论

就像你的电脑里有"文档""下载""桌面"，分类放，找起来才快。

### 2. 三层结构，不丢信息不费钱

L0：一句话摘要（像书名）
L1：结构化概要（像目录）
L2：完整记录（像全文）

OpenClaw 工作时默认只看 L0。

觉得相关才翻 L1。

真正需要细节才读 L2。

**省 token，省钱。**

### 3. 分类内去重

你告诉它"回复用中文"——偏好记忆。

你告诉它"上次用英文回复被骂了"——事件记忆。

两条都跟"语言"有关，但性质完全不同。

以前的方案会把它俩合并，乱套。

ePro 只在同类别内去重，井水不犯河水。

---

## 最关键的一点

**这条"记忆"不受上下文压缩影响。**

下次启动任务时，ePro 自动把偏好注入进去。

指令不丢，邮件不死。

---

## 字节跳动认证过

其实 ePro 的核心算法来自字节跳动的 **OpenViking**。

就是那个在字节内部打磨了无数次的 Agent 记忆分类体系。

不是拍脑袋想的，是真刀真枪在大量对话中验证过的。

韭二哥把它从字节拆出来，做了开源版本：

- Python 换成 TypeScript
- 向量数据库换成 LanceDB（本地运行，数据不跑路）
- 加了安全锁、防注入、113 个测试

**不终极，但免费。**
