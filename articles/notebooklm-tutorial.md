# 我让AI助手接管了NotebookLM，从此再也不用手动喂资料了

---

上个月想系统学OpenClaw。

打开文档站。

538个页面。

当场想放弃。

不是学不会。是光整理资料就能把人逼疯。

538个页面，一个个打开、复制、粘贴？

算了吧。

后来想到NotebookLM。

Google那个免费的AI知识助手，把资料丢进去，它能帮你生成播客、PPT、思维导图、测验题。还能基于资料回答问题，不瞎编。

问题是——我得先把538个页面"喂"进去。

NotebookLM没有API。不能写脚本批量导。只能手动粘贴URL。一条一条来。

538条。

我又想算了。

---

然后刷到一篇文章，说可以让OpenClaw直接操作NotebookLM。

AI指挥AI。我动嘴，它动手。

行，试试。

结果这一试，折腾了一下午。

---

第一步倒是秒过。跟AI助手说"帮我装NotebookLM skill"，几秒钟搞定。

然后卡了。

NotebookLM要登录Google账号。但我的AI助手在云服务器上，没屏幕没浏览器。

它说：你在自己电脑上登录，把认证文件发我。

我打开终端，看着它给我的命令——

pip install notebooklm-py

我愣了一下。

这玩意儿怎么跑？

别笑。我是搞内容的，不是程序员。终端对我来说就是个黑框框，里面全是咒语。

好在AI助手耐心，一步步教。装好了，跑了登录命令，浏览器弹出来，授权Google账号，点允许。

到这一步还挺顺。

然后它让我把认证文件发过去。

我翻了半天，找到个json文件，发了。

它说：哥，这是1月12号的，过期了。

啊？

原来我1月份就装过一次。登录了，没用，放了一个多月，Cookie早凉了。

重新登录，重新发。这次对了。

---

接下来才是爽的部分。

我说：把docs.openclaw.ai全站导进去。

它自己抓了sitemap，提取538个URL，开始一条条塞。

我去泡咖啡了。

回来一看，108个了。零失败。

又过了一会儿，到300个，被限流了。导太快，Google不让了。

它自己把速度从0.8秒一条降到2秒一条，继续跑。

我又去忙别的了。

最后538个，一个不少，全进去了。

然后我说：帮我生成一篇入门教程。

几分钟。一篇结构清晰的教程就出来了。安装、配置、自动化，全覆盖，每个点都有出处。

我自己翻文档？几天。

它？几分钟。

---

说说搞定之后能干嘛。

"帮我生成一期播客"——两个AI主持人聊你的资料，有节奏有追问。通勤路上听，比翻文档舒服。

"帮我出套测验题"——学完想验一下？它基于资料出题，比自己瞎背靠谱。

"帮我画个思维导图"——几百页的知识结构，一张图看完。

"帮我做套PPT"——自动提炼要点排版，还能用大白话让它改。

以前这些都能做。但前提是你得先手动把资料喂进去。

现在AI帮你喂。你只管提需求。

---

想自己搞的，完整步骤放这儿。我踩过的坑都标了。

**第一步：装skill**

跟你的OpenClaw说一句话就行：

"帮我安装NotebookLM skill"

它自己跑命令，几秒搞定。

**第二步：登录Google（最容易卡的地方）**

服务器没浏览器，登不了。得在自己电脑上搞。

打开终端，输入：

```
pip install notebooklm-py
notebooklm login
```

浏览器弹出来，登录Google，授权，关掉。完事。

⚠️ 坑：之前登录过但很久没用？Cookie会过期。别翻旧文件了，重新跑一次 `notebooklm login`。我在这浪费了半小时。

**第三步：把认证文件传到服务器**

认证文件在 `~/.notebooklm/storage_state.json`。

最简单的办法：直接在Telegram里把文件发给你的AI助手。它自己会放好。

或者命令行：`scp -r ~/.notebooklm/ root@服务器IP:~/`

**第四步：开始用**

最难的部分过去了。剩下的全交给AI。

"把xxx.com的文档全部导入NotebookLM"

它自己抓页面、批量导入、限流了自动降速。你该干嘛干嘛。

---

回头看，整个过程最多半小时。

最难的不是技术。是中间那几次想放弃的瞬间。

pip install不会跑？AI教你。Cookie过期了？重新登录。服务器没浏览器？本地搞完传过去。

每一步都不复杂。但叠在一起，就是容易让人打退堂鼓。

搞定之后的感觉就俩字：就这？

以前我是NotebookLM的搬运工。现在我是指挥官。

这个差别，比你想的大得多。
