# 内容创作者的 Agent 搭建实录：从素材焦虑到自动化工作流

## 一份基于真实生产环境的 OpenClaw 实战指南

我花两周时间搭建了一套"能听懂人话"的内容收集 Agent，把每天 2 小时的素材搜集压缩到 5 分钟。这篇文章把完整配置和踩坑经验分享出来，你可以直接复制使用。

---

## 一、为什么需要 Agent？

做公众号最痛苦的不是写，是找素材。

每天打开手机刷半小时，感觉看了很多，真正能用的没几条。再打开电脑搜一圈，还是不知道今天该写什么。一两个小时就这么没了，文章还没开始写。

这不是偷懒。是这件事本身就是无底洞：

```
找素材 → 判断适不适合 → 想切入点 → 整理备用
   ↑___________________________________________↓
                    明天重复
```

我发现自己不是在创作，是在做信息检索的苦力。

**Agent 的价值不是"更快"，是"你睡着的时候它在帮你干活"。**

## 二、核心原则：Agent 不是搜索引擎，是需要培训的助理

大部分人搭 Agent 的第一步就错了。

他们装好 OpenClaw，设置一个定时任务让 Agent "每天搜 AI 热点"，然后发现推过来的内容根本用不了——一堆模型参数对比、英文论文、技术圈黑话。

为什么？

**因为 Agent 不认识你。**

它推的是"AI 圈的热点"，不是"你的素材"。就像你请了一个助理，但没告诉他你是谁、你的读者是谁、你要写什么风格，他当然给你推一堆你根本用不上的东西。

我们的核心原则只有一条：

> **文件 = 事实来源。你不写进文件的东西 = Agent 从来不知道的东西。**

Agent 的"脑子"是空的，它启动时只认识磁盘上的文件。SOUL.md 和 USER.md 是它的"入职培训手册"，写得越详细，它上岗后越能干。

![先培训再上岗](agent-guide-img2.png)

这意味着：

- 花时间写清楚你的定位，比调试 100 次提示词都管用
- 你要像带新人一样，先培训再上岗
- 它的判断标准来自你写的文档，不是它的"常识"

## 三、四层记忆架构

我的系统分为四层，对应四种时间跨度的记忆：

| 层级 | 文件 | 生命周期 | 用途 |
|------|------|----------|------|
| **个人画像层** | SOUL.md, USER.md | 极少变化 | Agent 的人设 + 你的定位 |
| **工作记忆层** | _今日素材.md | 每天新建 | 当天收集的原始素材 |
| **素材库层** | 选题库/读者画像/爆款分析 | 持续积累 | 可复用的选题资产 |
| **决策层** | HEARTBEAT.md | 每次对话更新 | 当前任务状态和优先级 |

### 3.1 个人画像层（入职培训手册）

这是整个系统的底座。Agent 启动时首先读这两个文件，它们决定了 Agent "以什么身份、为谁、做什么"。

**SOUL.md —— Agent 的人设**

```markdown
# 我是谁

我是老里的内容助理，专门帮他找公众号素材、推选题。

## 我的工作方式
- 每天主动搜集 AI 相关内容
- 按老里的受众筛选，不是什么都推
- 推送时给出一句话说明"为什么这条值得写"
- 如果发现特别好的素材，主动建议选题角度

## 我的原则
- 不推技术论文、不推英文内容（除非特别重要且有中文解读）
- 优先推有真实故事的内容，不推纯概念
- 每次推 3-5 条，不够 5 条宁可少推也不凑数
- 判断标准是"老里的读者会不会想看"，不是"AI 圈热不热"

## 我熟悉的领域
- AI 工具怎么用（实操向）
- 普通人用 AI 的真实故事
- AI 对教育、工作、生活的影响
```

**USER.md —— 你的定位（重点！）**

```markdown
# 关于老里

## 我是谁
- 公众号博主，主号做 AI 科普，小号做工具教程
- 受众：普通 AI 爱好者、中小学教师、想提效的职场人
- 不是技术圈的人，是想用 AI 改变生活的普通人

## 我的读者画像
- 对 AI 感兴趣但没技术背景
- 想看到"我能怎么用"，不是"技术原理是什么"
- 喜欢真实故事，讨厌抽象概念
- 阅读场景：通勤、睡前，用手机看

## 我的内容方向
### 主号（深度）
- AI 热点事件的通俗解读
- 普通人用 AI 的真实案例
- AI 对教育/职场的影响分析

### 小号（工具）
- OpenClaw 配置教程
- AI 工具实操技巧
- 工作流自动化案例

## 我不写的内容
- 模型参数对比
- 技术原理深挖
- 纯英文内容
- 没有真实案例的概念解释

## 我的写作风格
- 口语化，像朋友聊天
- 从真实故事切入，再讲道理
- 短句为主，一段一个意思
- 读者是"你奶奶也能看懂"的水平

## 我的账号信息
- 主号：老里聊 AI（AI 科普深度）
- 小号：AI 实验室（工具教程实操）
- 更新频率：主号 2-3 篇/周，小号日更
```

**关键设计：写得越详细，Agent 推的内容越准。**

我第一次写 USER.md 只用了 5 行，结果 Agent 推的内容完全不对。后来扩充到上面的详细程度，效果立竿见影。

### 3.2 工作记忆层（每日素材）

每天新建的文件，记录当天收集的原始素材。

```
workspace/
└── 素材/
    ├── 2026-02-26.md
    ├── 2026-02-27.md
    └── 2026-02-28.md  ← 今天
```

**文件结构（Agent 自动生成）：**

```markdown
# 2026-02-28 素材收集

## 热点追踪
### OpenClaw 相关
- [ ] X 上有人分享 16 个 Agent 跑 13 个平台的案例
  - 来源：@wangray
  - 一句话：一人团队用 AI 自动化内容运营
  - 选题角度：普通人怎么从 0 搭第一个 Agent

### AI 教育
- [ ] 某中学老师用 AI 备课节省 2 小时/天
  - 来源：知乎热榜
  - 一句话：真实案例，有数据
  - 选题角度：教师群体的 AI 提效指南

## 读者反馈
- 小号粉丝留言问"Agent 怎么搭建"
- 主号评论区有人催更 OpenClaw 系列

## 今日选题决策
- [x] 写《普通人的第一个 AI Agent》（回应粉丝需求）
```

**关键设计：**

- Agent 每天 8 点自动写入，我只需要打开文件打勾确认
- 格式统一：`### 分类` + `- [ ] 条目`，便于扫描
- 包含"选题角度"建议，不是只给 raw data

### 3.3 素材库层（长期资产）

持续积累的可复用知识，不是临时素材。

```
workspace/
└── 素材库/
    ├── INDEX.md           # 导航表
    ├── 选题库.md          # 存着还没写的选题
    ├── 爆款分析.md        # 自己和他人的爆款复盘
    ├── 读者画像.md        # 积累的用户反馈
    └── 素材来源.md        # 可靠的信息源清单
```

**INDEX.md —— 导航表（Agent 启动时先读）**

```markdown
# 素材库导航

Agent 启动时先扫这个文件，按需加载具体内容。

## 选题库
| 状态 | 选题 | 来源 | 备注 |
|------|------|------|------|
| 待写 | 普通人第一个 Agent | 粉丝询问 | 高优先级 |
| 待写 | 学科教研组部署 | 教师需求 | 中优先级 |
| 已写 | OpenClaw 部署教程 | - | 爆款，可复盘 |

## 读者画像（关键！）
| 文件 | 说明 |
|------|------|
| [[教师群体]] | 中小学老师的痛点和需求 |
| [[职场人]] | 想提效的普通职场人画像 |

## 素材来源
| 来源 | 类型 | 可信度 |
|------|------|--------|
| X/Twitter | 海外热点 | 需筛选 |
| 知乎热榜 | 国内话题 | 高 |
| 即刻 | 圈内动态 | 中 |
```

**关键设计：**

- 带健康度指标（优先级、状态），Agent 扫描时能判断哪些信息可信
- 高优先级的选题和读者画像 = 核心知识，Agent 会重点参考

### 3.4 决策层（当前状态）

**HEARTBEAT.md —— 当前任务状态**

```markdown
# 当前状态

## 今日任务
- [x] 确认今日素材
- [ ] 写《普通人的第一个 AI Agent》
- [ ] 审稿：智慧校园那篇

## P0 优先级
| 任务 | 状态 | 截止时间 |
|------|------|----------|
| 第一 Agent 教程 | 写作中 | 今天 |
| 智慧校园审稿 | 待反馈 | 等老里 |

## 待确认
- DeepSeek 文章用哪版？（冷静版/燃文版）
```

**关键设计：**

- 每次对话结束时 Agent 更新，下次启动时先读
- 是 Compaction 后的"救生筏"——上下文被压缩后，从这里恢复状态

## 四、完整配置：从零搭建

### 4.1 文件结构总览

```
workspace/
├── SOUL.md                    # 个人画像层：Agent 人设
├── USER.md                    # 个人画像层：你的定位
├── HEARTBEAT.md               # 决策层：当前状态
│
├── 素材/                      # 工作记忆层
│   └── YYYY-MM-DD.md         # 每日素材（Agent 自动生成）
│
└── 素材库/                    # 素材库层
    ├── INDEX.md              # 导航表
    ├── 选题库.md
    ├── 爆款分析.md
    ├── 读者画像.md
    └── 素材来源.md
```

### 4.2 定时任务配置

在 Telegram 里跟 Agent 说：

> "帮我设置一个每天早上 8 点的定时任务，搜集今天的 AI 热点素材，按我的受众筛选后写入素材/今日日期.md，然后推送给我"

Agent 会自动创建 cron 任务。

**精确版任务描述（复制可用）：**

```
每天早上8点执行：

1. 搜索今天的 AI 相关热点，重点关注：
   - 普通人用 AI 的真实案例
   - AI 工具的新功能或新玩法
   - AI 对教育/职场的影响
   - OpenClaw/Agent 相关的实践经验

2. 筛选标准（按 USER.md 的画像）：
   - 有真实故事，不是纯概念
   - 适合非技术读者（中小学教师、普通职场人）
   - 中文内容优先
   - 判断依据：如果推给老里，他会觉得"这可以写"吗？

3. 写入格式（素材/YYYY-MM-DD.md）：
   ### 分类标签
   - [ ] 标题/一句话描述
     - 来源：
     - 一句话：为什么值得写
     - 选题角度：可以从什么角度切入

4. 完成后推送 Telegram：
   "今日素材已收集，共 X 条，请查看素材/2026-02-28.md"
```

### 4.3 手动触发测试

设置完后，先手动跑一次看看效果：

在对话框里说："现在帮我找一下今天的素材"

Agent 会立刻执行，把结果写到文件并推送给你。

**检查清单：**
- [ ] 推的内容符合你的定位吗？
- [ ] 选题角度建议有用吗？
- [ ] 一句话描述准确吗？

如果不对，回去改 SOUL.md 和 USER.md，改完再跑，直到满意。

## 五、一天的工作流

```
时间轴（一天）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

08:00  Agent 自动运行（cron 触发）
  │     ├─ 读取 SOUL.md + USER.md（了解任务）
  │     ├─ 读取 素材库/INDEX.md（了解资产）
  │     ├─ 搜集热点素材
  │     ├─ 按 USER.md 的画像筛选
  │     ├─ 写入素材/2026-02-28.md
  │     └─ Telegram 推送"今日素材已收集"
  │
08:05  我打开手机
  │     ├─ 看 Telegram 推送
  │     ├─ 打开素材文件扫一遍（5 分钟）
  │     ├─ 打勾确认要写的选题
  │     └─ 回复 Agent"ok"或"第 3 条换个角度"
  │
白天   正常写作
  │     素材已经在那里，不需要再花时间找
  │
晚上   复盘（可选）
  │     ├─ 今天写的文章数据如何？
  │     ├─ 有哪些可以存入选题库？
  │     └─ 更新素材库/读者画像.md
```

## 六、IP 调优：让 Agent 真正"懂你"

这是我最深的踩坑经验。

### 坑 1：USER.md 写得太简单

**错误示范：**
```markdown
# 关于我
- 公众号博主
- 写 AI 内容
```

**结果：** Agent 推一堆技术论文、英文新闻，完全不符合我的风格。

**正确做法：** 详细到"你奶奶也能看懂"这种具体描述。

### 坑 2：没有告诉 Agent 判断标准

**错误示范：**
```markdown
你的工作：每天搜集 AI 热点
```

**结果：** Agent 看到什么热就推什么，不管适不适合我。

**正确做法：**
```markdown
判断标准是"老里的读者会不会想看"，不是"AI 圈热不热"
```

### 坑 3：推的内容太多

**错误示范：** 让 Agent "多推几条，让我自己挑"

**结果：** 一次推 20 条，反而增加筛选负担。

**正确做法：**
```markdown
每次推 3-5 条，不够 5 条宁可少推也不凑数
```

![IP调优对比](agent-guide-img4.png)

### 进阶调优技巧

如果 Agent 推的内容还是不对，做这几件事：

**1. 发代表文章给它看**

把你写过的 3-5 篇代表文章发给它，说：

> "这是我写过的文章，你看看我的风格，以后按这个标准筛内容"

**2. 定期反馈**

- 推得准的：回复"这条很好，继续保持"
- 推得不准的：说"这条不符合，因为..."

Agent 会根据反馈调整，下次更准确。

**3. 更新读者画像**

在 `素材库/读者画像.md` 里记录粉丝的留言、反馈、需求：

```markdown
# 教师群体画像

## 痛点
- 备课时间长
- 不知道怎么用 AI
- 担心被 AI 替代

## 需求
- 想看到具体的备课案例
- 想知道其他老师怎么用
- 需要能直接套用的模板

## 反馈记录
- 2026-02-26：小号粉丝问"Agent 怎么搭建"
- 2026-02-27：主号评论区催更 OpenClaw 系列
```

Agent 每次启动都会读这个文件，慢慢就懂你的读者了。

## 七、从 1 个到 N 个

我现在的 Agent 不止做素材收集，还帮我：

- 写初稿（基于素材自动扩写）
- 排版（转成 HTML）
- 发布（推送到公众号草稿箱）

但这些都是后来慢慢加的。

**第一个 Agent 只做一件事：找素材。**

先把这一件事跑通，感受一下"睡着的时候它在帮你干活"是什么感觉。然后你自然会想要第二个、第三个。

## 九、写在最后

Agent 不难搭，难的是你得想清楚自己是谁。

这个问题想清楚了，Agent 才能真正帮到你。想不清楚，它只是一个更快的搜索引擎。

我的第一个 Agent 跑了 30 天，现在每天早上 8 点准时推送素材，准确率 80% 以上。剩下的 20% 我在 Telegram 里回复几句，它下次就会调整。

这就是普通人搭 Agent 的路径：**从 1 个开始，从小事开始，从认识自己开始。**
