# 写作方法论

## 核心认知
AI最大的问题不是写得不好，是写得太"对"了。宁可乱一点，也不要完美。
没有真实素材就不动笔，编出来的一定是AI味。

## 结构公式
故事 → 冲突 → 框架 → 升华

## 三大根因（AI味）
1. 套用AI默认科普模板，不是老里的方法论
2. 编造体验，没有真实细节
3. 缺少情绪曲线

## 爆款标题要素
1. 流量关键词（热点/大名词/情绪词）
2. 冲突悬念（反常识/场景化/前后反差）
3. 情绪钩子（贪婪/恐惧/懒惰/痛点）
4. 数字
5. 动词

## 平台定位（重要）

### 公众号
- 读者驱动力：**情绪满足**，被启发、被共鸣、感觉站在时代前沿
- 内容策略：敢"虚"，写趋势、写判断、写未来，格局感优先
- 不必每篇都给操作步骤，读完觉得"有格局"就够了
- 标题示例：《AI正在重塑教育，未来的老师会是什么样》

### 推特/X（李笔）
- 读者驱动力：**能力获取**，看完能做什么、能省多少时间
- 内容策略：开头直接给结论，中间全是可操作步骤，截图+命令+数字
- 情绪越少越好，干货越多越好
- 标题示例：《手把手教你用NotebookLM备课，5步省掉2小时》

### 同一选题的两种切法
同一个话题在两个平台要用两种写法：
- 公众号：卖**感觉**（启发、共鸣、格局）
- 推特：卖**技能**（步骤、截图、可复制）

### 同一选题的两种情绪版本（新）
同一个新闻也可以写两个情绪版本：
- 冷静版：先承认→再泼冷水→理性收尾（适合深度读者）
- 燃文版：情绪递进→民族自豪→热血收尾（适合流量传播）
- 两版同时给老里选，效率最高

## 写作协作流程（强制）
1. 先确认平台（公众号 or 推特），切换对应策略
2. 读 style-agent.md（每次）
3. 读 writing-guide.md（每次）
4. 问老里要真实素材
5. 写情绪大纲确认
6. 写初稿 → 过检查清单 → 老里审核
