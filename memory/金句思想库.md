## OpenClaw 踩坑实录（2026-03-02）

> 要做好龙虾，先要用最好的模型，最快的速度。

**已发现的核心问题：**

1. **LLM 调用太慢** — 即使用了 subAgent，仍有 session 死锁（工蜂和主 Agent 共享同一个 session 锁）
2. **历史消息不清空** — MessageCreateListener 处理历史消息，gateway 重启后消息不清空；Discord bot 重启后自动处理积压队列，导致很慢
3. **Pairing require 权限卡死** — 多 gateway、多 Agent 场景特别容易卡住
4. **SSRF 防护机制** — 有时会误拦截正常请求
5. **Opus 4.6 中转不稳定** — 考虑切换官方通道，等稳定后再切回

**结论：** 中转不稳定时，直接用官方通道，别硬撑。
