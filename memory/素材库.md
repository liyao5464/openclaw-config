
## [用户观点] OpenClaw安装避坑指南

**时间：** 2026-02-26
**来源：** 老里经验分享

### 核心观点

1. **不盲目买Mac mini**
   - "买台Mac mini跑OpenClaw才顺手"、"十分钟傻瓜式无脑安装"都是深坑
   - 对没有编程经验的新手小白而言，跟风买设备是大坑
   - 不缺钱的大佬另说，普通用户没必要

2. **本地安装容易踩坑**
   - 每个人的系统环境、版本、本地网络不同
   - 就算按教程复制命令，总会遇到各种奇葩报错
   - 小白根本无从下手解决

3. **让AI手把手教你（核心方法论）**
   - 不是看教程，而是直接把报错复制给AI（DeepSeek/GPT/Gemini都行）
   - 提供解决方案就行
   - 最小白的办法：直接给AI发这段话——
     
     > "我使用的是Win电脑，想通过安装WSL后，在Docker里安装OpenClaw。请给我一步步的最简指令，接下来遇到任何报错我都会直接发给你，请帮我解决"

4. **理性评估自己是否适合**
   - 任何产品都要实际使用后才知道是否适合
   - 也许你的idea在OpenClaw根本无法实现
   - 或者别人能做到的事情对你来说太难
   - **适合你的才是最好的**

5. **如果教程看了一堆还是不懂——建议放弃**
   - 即使叫别人帮你安装成功
   - 你大概率只会用默认版本
   - 后续高级用法肯定也无法学会（高级才是精华）

6. **不想折腾的解决方案**
   - 方案A：海鲜市场（找别人部署好的）
   - 方案B：腾讯云/阿里云一键部署（最省心）

### 金句摘录

- "不买设备，不盲目抄教程，让AI指导你用Docker做环境隔离，这才是普通小白玩转AI工具的办法"
- "适合你的才是最好的"
- "高级才是精华"

## [Twitter @sitinme] 让 AI Agent 变强的 4 个 Skills

**来源：** Twitter (https://twitter.com/sitinme/status/2028336497519456369)
**时间：** 2026-03-02

### 4 个实用 Skills

1. **find-skills** (Vercel)
   - 用 `npx skills find xxx` 搜索，一键安装更新
   - 价值：把"扩展能力"从翻 GitHub 变成稳定工作流

2. **skill-creator** (Anthropic)
   - 渐进式加载：先加载核心指令，资料按需加载
   - 自由度匹配：任务越脆弱指令越具体，越开放越留空间
   - **核心观点：Skill 不是写给人看的，是写给模型稳定执行的**

3. **Brainstorming Skill**
   - 先问清楚目标/约束/成功标准，再动手
   - 省返工：避免做到一半发现目标不清

4. **Systematic Debugging Skill**
   - 流程：复现→模式对比→最小测试→失败用例
   - 强迫一次只改一个变量，从"猜"变成"证据驱动"

### 金句
> "Skill 不是写给人看的，是写给模型'稳定执行'的。你把它当成'可复用的作业流程'，写出来的东西就会完全不一样。"
