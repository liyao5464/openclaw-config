## 6. 通用方法论

### 📋 发布检查清单
1. 确认发哪个号（主号/实验室）
2. 检查标题是否合适
3. 删掉备用标题
4. 封面提示词要简洁
5. 用 md2html-pro.js 转 HTML（实验室） 或 simple-md2html.js（主号）

### ✍️ 写作原则（融合 findyi 风格）

**结构公式：故事 → 冲突 → 框架 → 升华**
1. 开头用真实故事切入（读者私信/朋友案例/自身经历），100-200字
2. 放大痛点冲突，让读者感同身受
3. 结构化干货是核心，分点/分阶段/分层次，每点有案例支撑
4. 情怀金句收尾，引发共鸣和转发

**标题公式（任选其一）：**
- 悬念型：《那些真正会用AI的人，都在偷偷做这件事》
- 结果型：《普通人也能拥有8个AI助手？我试了3个月，真的可以》
- 对比型：《为什么你的AI总是失忆？因为你少做了这一步》
- 故事型：《一个中学老师，是怎么用AI把自己从重复工作中解放出来的》

**语言风格：**
- 口语化，不说教：用"我当时..."代替"你应该..."
- 短句为主，一句一段，适合手机阅读
- 数字要具体："省了2小时"比"节省时间"有力
- 自黑式幽默，接地气，不装
- 读者视角：痛点要踩在读者的真实焦虑上

**避免：**
- 官腔、说教味
- 照搬原文结构
- 开头就讲道理
- 结尾没有情感共鸣
