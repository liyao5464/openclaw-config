# 内容生产流水线 (Content Pipeline)

## 架构总览

```
你（选题）
    ↓
Writer Agent
  - 读取 style-agent.md 风格指令
  - 写初稿（含标题备选 + 配图提示词）
    ↓
QC Agent
  - 按 qc-agent.md 逐项检查
  - PASS → 进入生图
  - FAIL → 打回修改（最多2次）
    ↓
Image Agent (wan2.6-t2i)
  - 自动生成封面图 + 配图
  - 保存到 articles/ 目录
    ↓
老里（决策）
  - 收到：文章 + 封面图 + 配图
  - 确认发布 → 交给李微发布
  - 要修改 → 直接说哪里改
```

---

## 触发方式

### 方式一：直接告诉我（推荐）

发给小知了：
```
写文章：[选题]
```

例如：
```
写文章：DeepSeek V3 保姆级使用教程
写文章：AI帮老师批改作文，真实测评
写文章：OpenClaw学科教研组部署指南
```

我会自动：
1. 调用李笔写初稿
2. 跑QC检查
3. PASS了发你，FAIL了自动打回修改

### 方式二：直接找李笔

打开李笔的 Telegram Bot，发：
```
写一篇关于[选题]的文章，按照 style-agent.md 的风格
```

---

## DONE 标准（QC通过条件）

| 检查项 | 标准 |
|--------|------|
| 字数 | 1500-3000字 |
| 开头 | "大家好，我是老里" |
| 结尾 | 完整感恩三连 |
| 标题备选 | 至少3个 |
| 配图提示词 | 封面 + 至少2张 |
| 去AI味 | 风格评分 ≥ 35/50 |

全部通过 = 可发布通知老里

---

## 文件位置

| 文件 | 路径 | 用途 |
|------|------|------|
| 风格指南 | `workspace/style-agent.md` | Writer Agent 的写作规范 |
| 质检标准 | `workspace/qc-agent.md` | QC Agent 的检查清单 |
| 文章草稿 | `workspace/articles/` | 所有文章存放位置 |
| 李笔工作区 | `workspace-libi/` | 李笔的独立工作区 |

---

## 异常处理

- **QC连续2次FAIL** → 通知老里人工介入
- **字数严重不足（<1000字）** → 直接打回，不跑风格检查
- **开头/结尾缺失** → 直接打回，不跑风格检查
